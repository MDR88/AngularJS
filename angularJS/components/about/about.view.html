Primary Architecture
  • MVC - Model View Controller Architecture
  • Model - The model is where the data lives
  • View - The view is html and is what you see on the page
  • Controller - The controller is where the javascript works with the data that happens between the users front client end and the server back end of application
  • Routing - In all web applications it is very important to keep in mind that both the client and server side consist of routing http requests(request headers/request body). This basic request is usually the best starting point to understand the system as a whole. Look in the URL as you progress through your applications. Use F12 to open browser inspector and watch network HTTP requests going back and forth between the client and the server. Common headers are always going back and forth that you will come to recognize. Routing of requests occur on both the server and client side and tell each application what to do. Start trouble shooting by looking at the URL and the HTTP request sent to server. Remember in Angular that routing has to be imported separately and after the core library. It also has to be injected into the module at the start.
Core Controller Concepts
  • $scope - this allows a 1 to 1 biding between each view and its controller. Remember that it is in the router that we assign our view its controller. It is important in angular to remember that you can't use anything inside your controller functions that you don't pass into it. A common mistake is to add an Angular variable into a function like $http and then no request is going to the server. The reason is because in your controller when you put function($scope, $location, $http) these things needed to be passed in before they can be used in that function. This is true of all functions.
  • $rootScope - this variable allows us to bind across the entire application. This give us a way to have a global variable. With $scope you get only 1 to 1 with controller and view. RootScope though allows binding in any controller and in any view. If you find that you are using a lot of $rootScoped variables it is probably best you set up a service inside your application.
  • $location - This allows you to control the router and page change after you have performed some javascript in your application under the current view. An example of this is when you click on a button(ng-click) and pass in some data to a function and you want that views registered controller to perform some actions before going to the next view. At the end of your function you can use $location to change the page by going to different URL.
  • $http - this is how you make your GET, POST, PUT, DELETE, PATCH requests to your servers.
  • Services - this is the most common way to share data across all of your controllers and views. Basically you put common functions that your controllers can call to get data. An example might be something like the logged in users information. Each controller might only allow certain users access to certain data.
Core View Concepts
  • {{ scope.varibles }} - Use curly braces to render scope variables and ng-model data. This is Angular's default templating. This follows jQuery's older templating libraries like 'Handlebars' and 'Mustache' which both used curly braces as well to template out data on the view passed from the controller. Almost all languages have adopted similar approaches to template out data to the client's view.
  • ng-app - This must be in any Angular application to tell Angular where to begin taking over the application at. Because Angular is considered a SPA(Single Page Application) because entire pages are not re-rendered but instead Views are injected into the center of the index page, an ng-app is always required.
  • ng-view - This is where the router injects your views into the application.
  • ng-click - assign a click event on DOM(Document Object Model/Webpage) elements inside your .html view pages
  • ng-model - assigns 2-way data binding on form input elements so that you can use this to send data to server.
  • ng-show/ng-hide - This shows or hides elements on your webpage based on boolean(true/false) value. Because HTML is nested type language you can put an ng-show/ng-hide to scale what parts you want to show or hide. Entire pages or blocks of pages can be hidden and shown depeding on what tag you place this on.
  • ng-if - Similar to ng-show/ng-hide. This directive completely adds or removes elements from DOM based on boolean value as well. This is often helpful for placing two of the exact same size elements and it basically using it as an if/then statement allowing you to render one or the other in the DOM based on certain criteria.
  • ng-repeat - This is Angular's key way to iterate through collections. Collections in JavaScript are usually either object collections or array collections.
